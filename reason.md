# 학습 페이지 설계 근거 (reason.md)

> `/chat` 학습 페이지의 각 컴포넌트를 **어떤 교육학적 근거로** 넣었고, **왜 그 위치에** 배치했는지 정리.
> 설계 원칙 출처는 `LEARNING_PAGE_BRIEF.md` 참조 — 인지부하 감소(ITS/EDM 2024), 사고 스캐폴딩·자기설명(arXiv 2602.19303), 메타인지 사이클(arXiv 2511.04144), 소크라테스 대화(system_prompt.md).

## 이 버전의 핵심 방향
"답을 주지 않고 스스로 사고하게" 하되, **학습 도구가 별도 대시보드에 갇히지 않고 대화 흐름 속에서 자연스럽게 등장**하도록 설계했다. 화면을 3분할로 채우면 정보 밀도는 높지만 인지부하가 커지고 시선이 분산된다(EDM 2024). 그래서 **중앙 대화를 주인공으로 두고, 진행 현황만 좌측에 얇게, 학습 도구는 입력창 위·대화 흐름 안으로** 끌어왔다.

## 컴포넌트별 근거와 배치 이유

### 1. 학습 컨텍스트 헤더 (상단)
- **무엇**: 과목 배지(선택한 과목명) · 현재 개념 · 단계 배지(1단계·진단) · 목표 한 줄.
- **교육 근거**: 방향 상실(내가 지금 어디서 뭘 하는지 모름)은 그 자체로 인지부하다(ITS 연구). "지금 이 개념을, 이 단계에서" 를 항상 보이게 해 부하를 낮춘다.
- **왜 여기**: 화면 최상단은 시선의 기준점. 대화 내용이 바뀌어도 맥락 앵커가 고정돼 있어야 길을 잃지 않는다.

### 2. 진행 현황 — 진행바 + 선수개념 진단 맵 (좌측 사이드)
- **무엇**: 전체 진행률(%) 막대 + 4개 선수개념의 상태(점검 전/학습 중/숙달).
- **교육 근거**: 숙달학습·메타인지·동기. "무엇을 알고 무엇을 모르는지" 를 시각화하면 학습자가 스스로 목표 대비 위치를 점검(self-monitoring)한다.
- **왜 여기**: 좌측 주변시야에 **얇게**. 상시 참고는 되지만 대화를 방해하지 않는 거리. 큰 대시보드로 만들지 않은 이유 = 진단 결과는 '확인용'이지 '작업용'이 아니기 때문(부하 관리).

### 3. 학습 도구 3종 — 입력창 바로 위 (힌트 · 빈칸으로 연습 · 내 말로 설명)
- **교육 근거**:
  - **힌트 위계**: 좋은 튜터는 답을 설명하지 않고 *간접 힌트로 스스로 오류를 교정*하게 한다(arXiv 2602.19303). 그래서 정답이 아니라 관찰→연결→첫걸음 순의 점진 힌트, 그래도 막히면 그때만 튜터에게 유도.
  - **빈칸으로 연습(워크드예제→페이딩)**: 처음엔 풀이를 보여주고 점점 학습자가 빈칸을 채우게 해 스캐폴딩을 서서히 걷어낸다.
  - **내 말로 설명(자기설명)**: "왜 그런지 자기 말로 재진술" 이 학습효과가 가장 크다(self-explanation). 버튼이 자기설명 문장을 열어 유도한다.
- **왜 여기**: **막혔을 때 손이 가장 먼저 닿는 곳 = 입력창 옆**. 답을 쓰려다 막히는 바로 그 지점에 도구를 두어, 딴 데로 시선을 옮기지 않고 개입할 수 있게 했다.

### 4. 대화 흐름 안 인라인 요소 (힌트 카드 · 빈칸 카드 · 빠른 응답 칩)
- **무엇**: 도구를 누르면 힌트/빈칸이 **채팅 흐름 안에 카드로** 등장. 답변 뒤에는 상황별 선택지 칩.
- **교육 근거**: 자동 생성 선택지는 매번 타이핑하는 부하를 낮춘다(ITS). 힌트/빈칸이 별도 패널이 아니라 대화 안에서 나오면, **"튜터가 지금 건네주는 것"** 이라는 맥락이 유지돼 인지 흐름이 끊기지 않는다.
- **왜 여기**: 학습의 실제 사건(막힘→힌트, 답→다음 선택)은 대화에서 일어난다. 그 사건을 대화 타임라인 위에 남겨 학습 궤적이 그대로 기록되게 했다.

### 5. 강의 근거칩 (답변 아래 인라인)
- **무엇**: RAG로 검색된 출처를 "근거 · N쪽" 칩으로 답변 바로 밑에 표시(마우스오버 시 발췌).
- **교육 근거**: 출처 노출은 신뢰를 높이고 환각을 억제한다. AI 설명을 무비판 수용하지 않고 원문과 대조하게 만든다.
- **왜 여기**: 그 답변의 근거는 그 답변 옆에 있어야 즉시 대조된다. 별도 근거 패널로 빼면 대조 비용이 커진다.

### 6. "끝내기 →" 버튼 → 세션 성찰 (좌측 하단)
- **무엇**: 끝내기 버튼 → 성찰 다이얼로그(오늘 배운 것 / 아직 헷갈리는 것 / 자기평가) → 성찰을 대화로 정리하며 마무리.
- **교육 근거**: 자기조절학습은 계획→모니터링→**평가/성찰** 로 사이클이 닫혀야 완성된다(메타인지 스캐폴딩, arXiv 2511.04144). 세션 끝의 성찰이 학습을 장기기억으로 넘기고 다음 학습의 계획이 된다.
- **왜 여기**: 세션 종료는 '끝 지점'의 행동이므로 사이드바 맨 아래. 명시적 끝내기 버튼으로 "그냥 창을 닫는" 대신 **성찰하고 닫는** 습관을 유도한다.

### 7. 소크라테스 대화 (중앙, 핵심)
- **교육 근거**: 질문·반문으로 비판적 사고를 유도(system_prompt.md의 진단→backfill→유도 규칙과 일치). 나머지 컴포넌트는 모두 이 대화를 돕는 보조.
- **왜 여기**: 화면 중앙·최대 폭. 학습의 주무대이므로 시각적 우선순위 1순위.

## 요약 — 배치 철학
| 영역 | 넣은 것 | 한 줄 이유 |
|---|---|---|
| 상단 | 컨텍스트 헤더 | 방향 상실 방지(부하↓) |
| 좌측 | 진행 현황·진단맵 | 상시 참고, 대화는 방해 X |
| 중앙 | 소크라테스 대화 | 학습 주무대 |
| 입력창 위 | 힌트·빈칸·자기설명 도구 | 막히는 지점에 손 닿게 |
| 대화 흐름 안 | 힌트/빈칸 카드·선택칩·근거칩 | 학습 사건을 대화에 기록 |
| 좌측 하단 | 끝내기→성찰 | 메타인지 사이클 닫기 |
