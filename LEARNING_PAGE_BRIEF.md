# 학습 페이지(/chat) 교육적 개선 — 과제 브리프

> 각 워크트리(Codex · Cursor · hammerhead)에 동일 배포. 같은 목표를 각자 방식으로 구현 → 이후 비교·통합.
> 대상 파일: `index.html`(주), 필요 시 `app.py`. **기존 RAG 파이프라인·팔레트(forest/sage)·라우팅(/, /chat)은 유지.**

## 목표
`/chat` 학습 페이지를 "그냥 답 주는 챗봇"이 아니라 **학습과학 기반으로 교육 효과를 내는 구조**로 만든다. 핵심은 "답을 주지 않고, 스스로 사고하게 만드는 UI".

## 교육학 근거 (설계 원칙)
1. **인지부하 감소가 인터페이스의 핵심** — ITS 연구에서 화면이 사고를 방해하면 학습 효과가 떨어짐. 예: 자유 타이핑만 강요하지 말고 **자동 생성 선택지**로 답하게 하면 부하↓. 방향 상실(내가 어디쯤인지 모름)도 부하. [EDM 2024](https://educationaldatamining.org/edm2024/proceedings/2024.EDM-posters.78/) · [ITS 리뷰](https://arxiv.org/html/2507.18882v1)
2. **설명 전달이 아니라 사고를 스캐폴딩** — 좋은 튜터는 답을 설명해주는 게 아니라, **간접 힌트로 학생이 스스로 오류를 교정**하게 함. [대화형 튜터 종합](https://arxiv.org/html/2602.19303v1)
3. **자기설명(self-explanation)이 학습효과 최상** — 학생이 "왜 그런지 자기 말로 재진술"할 때 가장 많이 배움. 튜터가 이걸 유도해야 함. [동상]
4. **메타인지 사이클 완결(계획→모니터링→평가)** — 세션이 자기평가·성찰로 닫혀야 자기조절학습이 됨. [프로그래밍 교육 메타인지 스캐폴딩](https://arxiv.org/html/2511.04144v1)
5. **소크라테스/데블스 애드버킷** — 질문·반문으로 비판적 사고 유도. (우리 system_prompt.md와 일치)

## 추천 구조 — "학습 루프"를 화면에 구현
```
┌──────────────┬───────────────────────────┬──────────────────┐
│ 진행·진단 맵 │      소크라테스 대화       │ 컨텍스트 · 힌트  │
│ (좌 사이드)  │        (중앙, 핵심)        │   (우 패널)      │
└──────────────┴───────────────────────────┴──────────────────┘
```

### 넣을 컴포넌트 (우선순위 순)
1. **학습 컨텍스트 헤더** — 지금 학습 중인 강의·개념·단계 표시 (방향 상실 방지, 인지부하↓). *이미 lecture-badge 있음 → 확장.*
2. **선수개념 진단 맵 / 진행바** — "지금 뭘 알고 뭘 모르는지" 시각화 (숙달학습·메타인지·동기). 진단 대화 결과로 개념별 상태(부족/보통/숙달) 표시.
3. **힌트 위계(hint ladder) 버튼** — 막히면 "힌트 보기" → 점진적으로 세게 (스캐폴딩·부하 관리). **정답은 최후에만.**
4. **자기설명 프롬프트** — 답을 맞혀도 "왜 그런지 네 말로 설명해볼래?" 유도 (학습효과 최상).
5. **선택지 응답 옵션** — 매번 타이핑 대신, 가끔 **자동 생성 객관식**으로 답하게 (타이핑 부하↓, ITS 근거).
6. **워크드예제 → 페이딩** — 처음엔 예시 풀이를 보여주고, 점점 학생이 빈칸을 채우게.
7. **세션 성찰/자기평가** — 대화 끝에 "오늘 배운 것 / 아직 헷갈리는 것" 정리 (메타인지 사이클 완결).
8. **근거 표시** — RAG로 검색된 강의 근거 출처("강의 N쪽 기반") 노출 (신뢰·환각 억제).

## 제약 / 유지할 것
- 기존 RAG(`app.py`의 검색→생성) 로직 유지. UI가 데이터 붙일 수 있게 필요하면 `app.py`에 최소 엔드포인트만 추가.
- forest/sage 팔레트, Pretendard/Newsreader, 사이드바 구조 계승.
- 데모 안정성 우선: 정보이론 1강 프리셋으로 전체 학습 루프가 돌아가야 함.

## 결과물
- 개선된 `index.html`(+필요 시 `app.py`). 각자 브랜치(Codex/Cursor/hammerhead)에 커밋.
- 한 줄 요약: "어떤 컴포넌트를 왜 넣었는지" 근거를 커밋 메시지나 짧은 노트로.
