# 선수개념 진단 RAG 학습 튜터

답을 대신 풀어주는 대신, 강의자료를 읽고 **"이걸 이해하려면 뭐부터 알아야 하는지(선수개념)"**를 짚어 스스로 풀게 하는 AI 튜터.

> 한국외국어대학교 교육혁신원 · Better World 아이디어 공작소(교육&문화) 참가작
> 팀 **ARES** — 원종윤(팀장) · 오세창 · 윤태민 · 박현빈

## 왜

강의를 못 따라가는 이유는 그 과목이 어려워서가 아니라 **앞에 있어야 할 개념이 비어 있어서**인 경우가 많다. 그런데 이 결손은 수강 이력에 드러나지 않아 아무도 짚어주지 않는다. 공식 선수과목 표에는 없지만 실제로는 필요한 **"숨은 선수과목"** 이 문제다.

- 2024년 4년제 대학 중도탈락자 **10만 817명**, 증가 폭이 가장 큰 계열은 **인문사회계열**(2022년 대비 19.1% 증가).
- 한국외대 강의시간표를 조회해 보면 ELLT학과 「영어데이터분석을위한통계」, 경영학전공 「경영통계학」·「머신러닝」, 정치외교학과 「정치학연구방법론」, 행정학과 「행정계량분석」 등이 인문·사회계열 전반에 개설되어 있다. 그런데 **시간표에는 무엇을 먼저 들어야 하는지에 대한 안내가 없다.**
- 「AI정보이론」 1강은 제목이 통째로 **"확률 내용 복습"** 이다. 본론(엔트로피)에 들어가기 전에 교수가 이미 결손을 전제하고 있다는 뜻이다.
- 어느 전공 과제의 문항에는 **"간단하게라도 과정을 서술해주세요"** 라고 적혀 있다. 요구되는 것은 정답이 아니라 과정이므로, 정답만 주는 도구로는 이 요구를 채울 수 없다.

## 어떻게

**RAG(검색증강생성)** 로 강의자료를 근거 삼아 진단한다.

```
과목 선택 또는 강의자료 업로드 → 청킹 → 임베딩 → 벡터 검색(top-4)
                                                    ▼
                              LLM 진단 (정답 미제공, 앞 개념부터 확인)
                                                    │
                              결손 발견 시 ─────────┘
                                    ▼
                    backfill 선언("먼저 ○○가 필요해요")
                                    ▼
        유도(3단계 힌트 위계 · 오개념 선제 대응 · 빈칸 연습 = 워크드예제 페이딩)
                                    ▼
              원 개념으로 복귀 → 자기설명 요구 → 세션 성찰로 마무리
```

답변 아래에는 실제로 검색에 쓰인 **근거 단원(예: `AI정보이론 3강`)** 이 표시된다. 과목의 자료 단위에 따라 `강 / 주차 / 장` 으로 표기가 바뀐다.

## 지원 과목

과목마다 선수개념의 종류와 위계가 달라, 실제 강의를 분석해 과목별로 숨은 선수개념·오개념·진단 질문을 단원 단위 지식베이스로 따로 구축했다.

**지식베이스에는 강의자료 본문이 들어 있지 않다.** 강의자료의 저작권은 작성자(교수) 또는 출판사에 있고 수강생에게 재배포 권한이 없으므로, 각 지식베이스는 원문 대신 다음만 담는다.

- 각 단원이 다루는 개념의 목록 (사실 정보)
- 그 내용을 이해하는 데 필요한 **숨은 선수개념** (팀 분석)
- 학습자가 실제로 **틀리는 지점** (팀 분석)
- 정답 대신 되물을 **진단 질문** (팀 작성)
- 결손 발견 시 되돌아갈 **4단계 백필 경로**

특정 시험 문항의 정답·계산값·배점이나 교재 문제의 번호·수치도 담지 않는다. 근거칩은 원문 쪽수 대신 **단원까지** 가리키므로(예: `AI정보이론 3강`), 학습자는 본인이 보유한 강의자료에서 해당 단원을 찾아 대조하면 된다.

| 계열 | 과목 | 지식베이스 | 무너지는 지점 |
|---|---|---|---|
| 인문 | 언어학입문 | `lecture_ling.txt` (12단원 · 언어의 특성~언어습득) | 음소와 변이음의 구분 — 수식이 아니라 *관점*이 선수개념 |
| 사회 | 사회과학 연구방법론 | `lecture_survey.txt` (13단원 · 연구의 논리~질적연구) | 표본분포와 **표집분포**의 혼동 → p값이 끝까지 안 잡힘 |
| 사회 | 경영통계학 | `lecture_bizstat.txt` (13단원 · 대푯값·산포~두 집단 비교) | 자료의 분포와 **표본평균분포**의 혼동 |
| 사회 | 회계원리 | `lecture_acct.txt` (13단원 · 회계등식~주식회사의 자본) | 발생주의 시점 판단 — 영어 용어 부담과 겹침 |
| 이공 | AI정보이론 | `lecture_info.txt` (13단원 · 확률 복습~채널 부호화) | 조건부확률·기댓값 결손이 엔트로피에서 드러남 |
| 이공 | 머신러닝 | `lecture_ml.txt` (11단원 · 선형대수~SVM) | 선형대수·확률·다변수 미적분에 동시에 걸쳐 있음 |

준비된 과목이 없으면 시작 화면(`/start`)에서 강의자료(PDF·필기)를 올려 바로 진단할 수 있다.
프리셋 과목은 임베딩을 사전 계산해 `*_vecs.npy`로 보관하므로 첫 응답이 **약 60초 → 2.2초**로 줄었다.

## 빠른 시작

```bash
git clone https://github.com/zibb03/hufs-ares-prereq-tutor.git
cd hufs-ares-prereq-tutor
pip install -r requirements.txt

# .env 파일을 새로 만들고 (git에 안 올라감, 각자 발급)
# aistudio.google.com 에서 무료 발급
echo "GEMINI_API_KEY=여기에_본인_키" > .env

uvicorn app:app --reload
# → http://localhost:8000
```

`python app.py` 만 실행하면 API 키 없이 검색 로직 셀프체크만 돌릴 수 있다.

## 기술 스택

Python · FastAPI · Google Gemini API(`gemini-2.5-flash` 생성 · `gemini-embedding-001` 임베딩) · Numpy(코사인 유사도) · PDF.js(클라이언트 텍스트 추출) · Tailwind CSS

## 파일 구성

| 파일 | 역할 |
|---|---|
| `app.py` | RAG 파이프라인(청킹·임베딩·검색·생성) + 과목 레지스트리 + FastAPI 서버 |
| `system_prompt.md` | 튜터 행동 규칙(정답 미제공·진단·backfill·유도) |
| `landing.html` | 소개 페이지 (`/`) — 문제 정의·지원 과목·작동 원리·근거 |
| `select.html` | 과목 선택·업로드 페이지 (`/start`) |
| `index.html` | 학습 화면 (`/chat`) — 대화 내 힌트·빈칸·근거칩·성찰 |
| `lecture_*.txt` | 과목별 선수개념 지식베이스 (위 표 참조) |
| `lecture_*_vecs.npy` | 프리셋 과목의 사전 계산 임베딩 |
| `COMPONENTS.md` | 학습 화면 컴포넌트 8종과 교육적 근거 |
| `MISSION.md` | 프로젝트 목적·설계 명세 |
| `DEV.md` | 개발 진행 기록·TODO (작성 당시 상태 기준) |
| `SUBJECT_DESIGN_TEMPLATE.md` | 새 과목을 추가할 때 채우는 템플릿 |

## 참고

- 데모 문서는 서버 요청마다 새로 읽으므로(`FileResponse`), `index.html` 수정 후 새로고침만 하면 반영된다.
