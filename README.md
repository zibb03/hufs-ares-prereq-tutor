# 선수개념 진단 RAG 학습 튜터

답을 대신 풀어주는 대신, 강의자료를 읽고 **"이걸 이해하려면 뭐부터 알아야 하는지(선수개념)"** 를 짚어 스스로 풀게 하는 AI 튜터.

> 한국외국어대학교 교육혁신원 · Better World 아이디어 공작소(교육&문화) 참가작
> 팀 **ARES** — 원종윤(팀장) · 오세창 · 윤태민 · 박현빈

## 왜

강의를 못 따라가는 이유는 그 과목이 어려워서가 아니라 **앞에 있어야 할 개념이 비어 있어서**인 경우가 많다. 그런데 이 결손은 수강 이력에 드러나지 않아 아무도 짚어주지 않는다. 공식 선수과목 표에는 없지만 실제로는 필요한 **"숨은 선수과목"** 이 문제다.

- 2024년 4년제 대학 중도탈락자 **10만 817명**, 증가 폭이 가장 큰 계열은 **인문사회계열**(2022년 대비 19.1% 증가).
- 통계·데이터 과목이 인문·사회계열 전반에 개설되어 있는데 **공식 선수과목 지정이 없는** 경우가 많다.
- 어떤 과목은 1강 전체가 선수 내용 복습으로 편성되어 있다. 즉 교수자가 이미 결손을 전제하고 있다는 뜻이다.
- 한 과목의 중간고사 채점 기준 첫 줄은 **"모든 문제에 설명 없이 답만 있을 경우 0점"** 이었다. 교수자가 요구하는 것은 정답이 아니라 설명이므로, 정답만 주는 도구로는 이 요구를 채울 수 없다.

## 어떻게

**RAG(검색증강생성)** 로 강의자료를 근거 삼아 진단한다.

```
과목 선택 또는 강의자료 업로드 → 청킹 → 임베딩 → 벡터 검색(top-4)
                                                    ▼
                              LLM 진단 (정답 미제공, 앞 개념부터 확인)
                                                    │
                              결손 발견 시 ─────────┘
                                    ▼
                    backfill 선언("먼저 ○○가 필요해요")
                                    ▼
        유도(3단계 힌트 위계 · 오개념 선제 대응 · 빈칸 연습 = 워크드예제 페이딩)
                                    ▼
              원 개념으로 복귀 → 자기설명 요구 → 세션 성찰로 마무리
```

답변 아래에는 실제로 검색에 쓰인 **강의 근거(예: `AI정보이론 3강 · 6쪽`)** 가 표시된다. 과목의 자료 단위에 따라 `강 / 주차 / 장` 으로 표기가 바뀐다.

## 이 저장소에 강의자료 원문이 없는 이유

이 프로젝트는 실제 대학 강의자료를 근거로 검증했지만, **강의자료의 저작권은 작성자(교수) 또는 원서 출판사에 있다.** 수강생이 자료를 제공받은 것은 이용 허락이며 재배포 권한이 아니다. 따라서 이 공개 저장소에는 강의 원문을 포함하지 않는다.

- **포함**: 코드, 화면, 그리고 **팀이 직접 작성한 선수개념 지식베이스 및 진단 부록**
- **제외**: 실제 강의자료 원문 및 그로부터 생성된 임베딩

원문을 사용하는 과목(AI정보이론·경영통계학·회계원리)은 `lecture_*_appendix.txt` 형태로 **팀이 작성한 진단 자료만** 공개한다. 각 부록에는 해당 과목에서 학습자가 실제로 막히는 지점과 되물을 질문이 정리되어 있으며, **특정 시험 문항의 정답·계산값·배점이나 교재 문제의 번호·수치는 담지 않았다.**

`app.py`는 지식베이스 파일이 없는 과목을 자동으로 목록에서 숨기므로, 원문 없이 클론해도 그대로 실행된다. 본인 강의자료를 직접 올려 쓰는 경로가 기본 기능으로 들어 있다.

## 지원 과목

| 계열 | 과목 | 이 저장소 포함 여부 | 무너지는 지점 |
|---|---|---|---|
| 인문 | 언어학입문 | 지식베이스 포함 (팀 작성) | 음소와 변이음의 구분 — 수식이 아니라 *관점*이 선수개념 |
| 사회 | 사회과학 연구방법론 | 지식베이스 포함 (팀 작성) | 표본분포와 **표집분포**의 혼동 → p값이 끝까지 안 잡힘 |
| 사회 | 경영통계학 | 부록만 포함 (원문 제외) | 자료의 분포와 **표본평균분포**의 혼동 |
| 사회 | 회계원리 | 부록만 포함 (원문 제외) | 발생주의 시점 판단 — 영어 용어 부담과 겹침 |
| 이공 | AI정보이론 | 부록만 포함 (원문 제외) | 조건부확률·기댓값 결손이 엔트로피에서 드러남 |
| 이공 | 머신러닝 | 지식베이스 포함 (팀 작성) | 선형대수·확률·다변수 미적분에 동시에 걸쳐 있음 |

## 빠른 시작

```bash
git clone https://github.com/zibb03/hufs-ares-prereq-tutor.git
cd hufs-ares-prereq-tutor
pip install -r requirements.txt

# .env 파일을 새로 만들고 (git에 안 올라감, 각자 발급)
# aistudio.google.com 에서 무료 발급
echo "GEMINI_API_KEY=여기에_본인_키" > .env

uvicorn app:app --reload
# → http://localhost:8000
```

`python app.py` 만 실행하면 API 키 없이 검색 로직 셀프체크만 돌릴 수 있다.

새 과목을 추가하려면 `SUBJECT_DESIGN_TEMPLATE.md` 를 채워 `lecture_<id>.txt` 로 저장하고 `app.py` 의 `PRESETS` 에 등록한다. 프리셋 과목은 임베딩을 사전 계산해 `*_vecs.npy` 로 보관하면 첫 응답이 **약 60초 → 2.2초**로 줄어든다.

## 파일 구성

| 파일 | 역할 |
|---|---|
| `app.py` | RAG 파이프라인(청킹·임베딩·검색·생성) + 과목 레지스트리 + FastAPI 서버 |

| `lecture_*.txt` | 팀이 작성한 과목별 선수개념 지식베이스 |
| `lecture_*_appendix.txt` | 원문을 공개하지 않는 과목의 진단 부록 (팀 작성) |
