# MISSION.md — 선수개념 진단형 RAG 학습 튜터 (설계·아키텍처)

> ARES 팀 · Better World 아이디어 공작소(교육&문화) 발표 시연용 MVP
> ※ 구현 방향: 학습자 강의 콘텐츠를 지식베이스로 삼는 **RAG(검색증강생성)** 파이프라인. **Python + Gemini(무료)**.

## 프로젝트 목적
답을 대신 풀어주는 기존 AI와 달리, 학습자가 올린 **강의 콘텐츠를 검색(RAG)**해 **'선수개념 결손'을 진단**하고, 과목 경계를 넘어 보완(backfill)하며, **답 대신 질문**으로 스스로 이해에 도달하게 돕는 유도형 AI 튜터.

## 대상 사용자
선수개념 결손에 특히 취약한 학습자 — **편입생 / 무전공(자유전공) 입학생·비전공 전환자 / 외국인 유학생(한국어 부담과 결손의 이중 장벽) / 휴학·복학생 / 사교육 접근이 어려운 학습자.** 이들은 모두 수강 이력상 '선수과목을 이수한 학생'으로 기록되므로, 이력이 아니라 이해도를 직접 진단해야 한다.
**데모 과목: 언어학입문 · 사회과학 연구방법론 · AI정보이론 · 머신러닝** (인문·사회·이공 4과목)

## RAG 파이프라인
문서 → **청킹** → **임베딩** → **검색(top-k)** → **LLM 생성(시스템프롬프트 주입)** → 응답
- 청킹/임베딩/검색: `app.py` (in-memory numpy 코사인 · `# ponytail: 벡터DB는 코퍼스 커지면`)
- 임베딩·생성 LLM: **Gemini**(`gemini-embedding-001` 3,072차원 + `gemini-2.5-flash`), 키는 서버 환경변수
  ※ 초기에 지정한 `text-embedding-004`는 응답하지 않아(404) 모델 목록 조회로 확인해 교체함
- 프리셋 과목은 임베딩을 사전 계산해 `*_vecs.npy`로 보관 → 첫 응답 약 60초 → 2.2초
- 진단·backfill·유도 규칙: `system_prompt.md` (systemInstruction으로 주입)

## 기술 스택
- **Python** (FastAPI + numpy, LLM 호출은 urllib REST — 의존성 최소)
- **LLM:** Gemini API (Google AI Studio 무료 티어). `GEMINI_API_KEY` 서버 보관.
- **프론트:** `index.html` (Tailwind CDN 채팅, `/api/chat` 호출)

## 핵심 기능 (데모)
1. 과목 선택(`/start`) 또는 강의자료 업로드 → 청킹·임베딩 인덱싱
2. 진단 대화(정답 미제공) → 선수개념 결손 감지
3. backfill 선언("먼저 ○○ 필요") → 3단계 힌트 위계·빈칸 연습으로 유도 → 원 개념 복귀
4. 검색 근거 노출(예: `AI정보이론 3강 · 6쪽`)로 환각·오진 방지
5. 자기설명 요구 → 세션 성찰(오늘 배운 것 / 헷갈리는 것 / 설명 가능 수준)로 마무리

## 성공 기준
- 4개 과목 각각에서 **검색→진단→backfill→유도→복귀→성찰** 흐름이 실제 RAG+LLM 대화로 작동
- "정답 바로 안 줌" 원칙 유지 · 발표 영상 시연

## Non-goals
- 로그인·회원 DB·결제 없음 (계정 기반 누적 지식지도는 **로드맵**)
- 진행률은 **대화 기준 시각화**이며 성취 판정이 아님 (구조화된 진단 출력·확인 문제 채점은 로드맵)
- 이력은 브라우저 저장이라 기기 간 미연동 · 힌트/빈칸은 저작된 과목에만 노출(업로드 과목은 동적 생성 필요)

## Key Concepts
- **RAG:** 강의 콘텐츠를 검색해 LLM 답변 근거로 삼는 구조
- **숨은 선수과목:** 공식 선수과목 표엔 없으나 실제 필요한 개념(정보이론의 확률·통계)
- **backfill:** 결손 선수개념을 과목 경계 넘어 먼저 채우기
- **유도형(Socratic):** 답이 아니라 질문·힌트로 스스로 도달
